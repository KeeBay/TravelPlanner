4. Kockázatkezelés

Kockázat Hatás Kezelés módja
Szerveroldali hiba / Offline API (500) Magas Try-catch blokk és hibaüzenet ("❌ Hiba történt a küldés közben!").
Szkript alapú támadás (XSS) Magas A kimenet textContent-ként való megjelenítése a innerHTML helyett.
SQL Injection kísérlet Magas JSON formátumú adatküldés és backend oldali paraméterezett lekérdezések.
Hibás kapcsolati adatok Közepes HTML5 natív validáció és kliensoldali ellenőrzés a hálózati forgalom előtt.
Token ellopása / Munkamenet hiba Közepes Token tárolása localStorage-ban, validálás minden védett kérésnél.
Űrlap többszöri beküldése Alacsony Frontend eseménykezelő blokkolás (preventDefault).

5. Elfogadási kritériumok

  • A felhasználó csak érvényes hitelesítő adatokkal férhet hozzá a rendszerhez.
  • A sikeres bejelentkezés után a JWT token fizikailag jelen van a böngésző tárolójában.
  • A beviteli mezők ellenállnak az alapvető injekciós támadásoknak.
  • Hálózati hiba esetén a felhasználó nem marad információ nélkül (hibaüzenet megjelenik).
